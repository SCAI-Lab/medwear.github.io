Contribute to MEDWEAR

MEDWEAR is a community-driven project that brings together researchers, industry experts, and developers to create open standards everyone can use. We welcome contributions to the schemas, ROS messages, platform, and documentation.

Choose what to contribute to

JSON Schemas

Add or improve physiological signal schemas based on IEEE 1752 and Open mHealth. Propose new signal types, fix definitions, or improve documentation.

ROS 2 Messages

Contribute to the healthcare_msgs package — add new .msg definitions, improve existing ones, or extend support for new ROS distros.

MEDWEAR Platform

Contribute to the data collection platform and MED-API — bug fixes, new features, deployment improvements, or integration with new storage backends.

Conversion & Streaming Tools

Improve conversion scripts, add support for new file formats, or extend the streaming libraries for MQTT, UDP, WebSockets, or BLE.

How to Contribute

  1. Fork the relevant repository on GitHub (links above).
  2. Make your changes or additions, including detailed justifications and documentation.
  3. Submit a Pull Request describing your changes clearly.
  4. We will review your submission and provide feedback.

We encourage active participation and welcome discussions on issues and feature requests.
