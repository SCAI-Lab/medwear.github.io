Quick Start

JSON Schema structure

Each MEDWEAR schema defines a data structure for a physiological signal. Here is a minimal ECG data example: 

{
  "schema_id": "org.medwear.biosignal.ecg",
  "version": "1.0",
  "data": {
    "ecg": [0.12, 0.15, 0.11],
    "sample_rate": 250,
    "units": "mV"
  },
  "metadata": {
    "device_id": "device1234",
    "timestamp": "2025-06-30T15:30:00Z"
  }
}

Validating data in Python

Use the jsonschema library to validate your data against a MEDWEAR schema:

import json
from jsonschema import validate

with open('ecg_schema.json') as f:
    schema = json.load(f)

with open('ecg_data.json') as f:
    data = json.load(f)

validate(instance=data, schema=schema)
print("Data is valid.")

Integration Guides

JSON Schemas

MEDWEAR schemas extend Open mHealth to support raw, high-frequency sensor formats. Download individual schemas from the Schemas page or browse the full set on GitHub.

ROS 2 Messages

ROS 2 .msg definitions are available for integration with healthcare robots and assistive devices. The healthcare_msgs package supports ECG, BCG, PPG, EEG, EDA, IMU, HR, and HRV.

View the ROS 2 message package on GitHub

Conversion Tools

Command-line tools and notebooks are available to convert CSV, JSON, and raw wearable files into MEDWEAR-compliant format.

Explore conversion tools on GitHub

MED-API

For server-side ingestion, querying, and management of wearable data, see the MED-API documentation. It provides a REST API backed by InfluxDB, PostgreSQL, and MinIO.
